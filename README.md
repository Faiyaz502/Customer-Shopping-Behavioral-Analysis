## 📌 Project Overview
This project simulates an industry-standard data analytics workflow, translating **5,000+ unstructured consumer records** into strategic business intelligence. By integrating **Python** for ETL, **SQL** for relational modeling, and **Power BI** for visualization, the engine identifies high-value customer segments and actionable loyalty drivers to mitigate churn.

---

## 🏗️ System Architecture
The pipeline follows a modular 4-phase architectural approach:

1.  **Data Engineering (ETL):** Automated cleaning and feature engineering using Python.
2.  **Relational Modeling:** Data migration to PostgreSQL for ACID-compliant structured querying.
3.  **Advanced Analytics:** Utilizing SQL Window Functions & CTEs for deep-dive behavioral insights.
4.  **Business Intelligence:** Developing interactive dashboards and stakeholder-ready reports.

## 🚀 Key Features & Implementation

### 1. Automated ETL Pipeline (Python)
* **Integrity:** Cleaned 5,000+ raw records, resolving null values and type inconsistencies.
* **Feature Engineering:** Calculated **Customer Lifetime Value (CLV)** and **Purchase Frequency** to enrich the consumer profile.
* **Outcome:** Achieved **100% data integrity** and eliminated manual preparation time.

### 2. Relational Modeling & SQL Analytics
* **Structure:** Designed a **3rd Normal Form (3NF)** schema to simulate production-grade business environments.
* **Complex Logic:** Authored scripts utilizing **Window Functions** and **CTEs** to isolate "Power Users" and churn risk indicators.
* **Optimization:** Implemented indexing to enable **millisecond-latency** search across multi-table joins.

### 3. Strategic BI Dashboard (Power BI)
* **Interactivity:** Dynamic slicers for demographics, spending behavior, and regional performance.
* **Key Discovery:** Identified a **12% correlation** between service latency and customer churn.
* **Outcome:** Created a "Single Source of Truth" for executive-level KPI monitoring.
